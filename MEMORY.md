# Memory

This file stores persistent learnings across Claude Code sessions. It loads into Claude's context automatically.

## How to Use

- Add entries with `[LEARN:workflow]`, `[LEARN:domain]`, or `[LEARN:technical]` tags
- Keep entries concise (1-2 lines each)
- Remove outdated entries when corrected
- This file stays under 200 lines

## Memory System

This repository uses a two-tier memory system:

1. **MEMORY.md** (this file, committed to repo) — Generic learnings that sync across machines via git
2. **`.claude/state/personal-memory.md`** (gitignored, local only) — Machine-specific and user-specific learnings

See `.claude/rules/meta-governance.md` for the complete memory management protocol.
