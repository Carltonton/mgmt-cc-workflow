# PDF Extraction with MinerU

Extract PDF content to Markdown format using the official MinerU toolkit. Supports PDF, images, DOCX, PPTX, and XLSX files.

## Installation

```bash
pip install mineru
```

For API server functionality:

```bash
pip install "mineru[api]"
```

## Configuration

### Model Source

MinerU defaults to `huggingface`. If inaccessible, switch to `modelscope`:

```bash
export MINERU_MODEL_SOURCE=modelscope
```

For local models, set `MINERU_MODEL_SOURCE=local` and configure paths in `mineru.json`.

### Configuration File (`mineru.json`)

Create `mineru.json` in your user directory (auto-generated by `mineru-models-download`):

```json
{
  "latex-delimiter-config": {
    "single": "$",
    "double": "$$"
  },
  "llm-aided-config": {
    "api_key": "your_api_key",
    "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
    "model": "qwen3-next-80b-a3b-instruct",
    "enable": false
  },
  "models-dir": {
    "pipeline": "/path/to/pipeline/models",
    "vlm": "/path/to/vlm/models"
  }
}
```

**Options:**

- `latex-delimiter-config`: Custom LaTeX formula delimiters (default: `$`)
- `llm-aided-config`: LLM-aided heading hierarchy (OpenAI-compatible)
- `models-dir`: Local model storage paths

## Usage Modes

### 1. CLI (Command-Line Interface)

**Basic extraction:**

```bash
# macOS: always prefix with CPU mode (prevents screen corruption)
MINERU_DEVICE_MODE=cpu mineru -p <input_path> -o <output_path>

# Linux: GPU acceleration is safe
mineru -p <input_path> -o <output_path>
```

- `<input_path>`: File or directory (PDF/images/DOCX/PPTX/XLSX)
- `<output_path>`: Output directory

**Connect to existing API server:**

```bash
mineru -p paper.pdf -o output/ --api-url http://127.0.0.1:8000
```

**Preload VLM models:**

```bash
mineru -p paper.pdf -o output/ --enable-vlm-preload true
```

### 2. API Server

Start the FastAPI server:

```bash
mineru-api --host 0.0.0.0 --port 8000
```

**Endpoints:**

- `GET /health` - Service status and configuration
- `POST /tasks` - Async task submission
- `POST /file_parse` - Synchronous parsing
- `GET /tasks/{task_id}` - Task status
- `GET /tasks/{task_id}/result` - Task result

**API Documentation:** http://127.0.0.1:8000/docs

**Async submission:**

```bash
curl -X POST http://127.0.0.1:8000/tasks \
  -F "files=@paper.pdf" \
  -F "return_md=true"
```

**Synchronous parsing:**

```bash
curl -X POST http://127.0.0.1:8000/file_parse \
  -F "files=@paper.pdf" \
  -F "return_md=true" \
  -F "response_format_zip=true"
```

**Environment variables:**

- `MINERU_API_TASK_RETENTION_SECONDS` - Task retention time (default: 86400)
- `MINERU_API_TASK_CLEANUP_INTERVAL_SECONDS` - Cleanup interval

### 3. Gradio WebUI

Interactive browser interface:

```bash
mineru-gradio --server-name 0.0.0.0 --server-port 7860
```

Access at http://127.0.0.1:7860

### 4. Router (Multi-GPU)

For multi-GPU deployments:

```bash
mineru-router --host 0.0.0.0 --port 8002 --local-gpus auto
```

Aggregates multiple `mineru-api` instances with unified endpoints.

## Output Format

MinerU generates:

- `.md` - Markdown content with preserved structure
- `.json` - Structured content with layout information
- Images embedded as base64 or separate files

## Examples

```bash
# Single PDF extraction
mineru -p docs/paper.pdf -o references/md/

# Batch extraction from directory
mineru -p docs/papers/ -o references/md/

# Using API server mode
mineru-api --host 0.0.0.0 --port 8000

# Interactive WebUI
mineru-gradio --server-name 0.0.0.0 --server-port 7860
```

## macOS GPU & Memory Safety

> **WARNING**: On Apple Silicon, MinerU auto-selects MPS (Metal GPU) for inference,
> which shares unified memory with the display server. This can cause **screen
> corruption (花屏)**, flickering, or system freezes on large PDFs.

### Mandatory: Force CPU Mode on macOS

**Always** run MinerU with these environment variables on Mac:

```bash
export MINERU_DEVICE_MODE=cpu
export MPS_DISABLE=1
export CUDA_VISIBLE_DEVICES=""
mineru -p <input.pdf> -o <output/>
```

Or inline:

```bash
MINERU_DEVICE_MODE=cpu MPS_DISABLE=1 mineru -p paper.pdf -o output/
```

### Optional: Memory Limiting

For large documents, restrict virtual memory to avoid system-wide pressure:

```bash
# Bash: limit to 4 GB virtual memory
ulimit -v 4194304
MINERU_DEVICE_MODE=cpu mineru -p large_doc.pdf -o output/
```

### Why This Happens

Apple Silicon uses **unified memory** — the GPU and CPU share one memory pool.
When MinerU loads ML models via MPS, it competes with WindowServer (the display
 compositor) for the same memory, causing display artifacts. Forcing CPU mode
avoids this entirely.

## Notes

- CLI automatically attempts CUDA/MPS acceleration on Linux/macOS — **override on Mac** (see above)
- For Windows CUDA support, install appropriate torch version
- Output preserves structure, tables, formulas, and images
- Tasks retained for 24 hours by default (adjustable via env vars)
